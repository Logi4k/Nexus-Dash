# Nexus App Audit

Date: 2026-04-08

Scope:
- Routes: Dashboard, Market, Journal, Prop Accounts, Expenses, Debt, Tax, Investments, Ideas
- Modes: desktop + mobile, dark + light
- Method: browser review plus automated overflow checks

## Priority findings

1. Dashboard chart implementation is structurally fragile.
   The custom monthly chart in `src/pages/Dashboard.tsx` uses manual bar heights, nowrap month labels, flex-driven column widths, and no hard clipping on the chart region. With real data, that can leak labels or bars beyond the card edge.

2. Mobile dashboard header and hero are too dense.
   Title, date chip, decorative glows, and horizontal account pills all compete in a narrow width. The page does not hard-scroll horizontally, but several internal regions are under overflow pressure.

3. Mobile investments header is the weakest header layout.
   Status pill, sync text, sync button, and `Add Holding` fight for one row.

4. Mobile prop header is also too tight.
   `Record Payout` and `Add Account` beside the page title compress the top section.

5. Tax has real text-fit issues on mobile.
   Supporting helper copy is currently truncating where it should wrap.

## Theme findings

1. Dark mode is closer to premium than light mode.
2. Light mode loses hierarchy fastest.
3. Section identity should come from accent only, not different surface behavior per page.

## Animation opportunities

1. Route transitions
   Add a short fade + y-shift on page changes around the outlet.

2. Buttons
   Standardize hover tint, press scale, and focus-ring interpolation across primary and ghost buttons.

3. Active navigation
   Animate active nav background and indicator between routes instead of snapping.

4. Cards
   Add restrained first-paint stagger only to major dashboard sections, not every list item.

5. Charts and counters
   Unify durations for number changes, progress bars, and chart selection states.

## Recommended order

1. Pick one color direction from the mockups.
2. Reset dark/light surfaces and typography globally.
3. Fix dashboard chart structure.
4. Reflow mobile page headers.
5. Add shared motion patterns.
