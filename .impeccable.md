# Nexus — Impeccable Design Context

## Design Context

### Users
Prop traders and self-funded traders managing their financial life in one place. UK-based (GBP primary, USD secondary). They use this app daily — before markets open, mid-session, and for end-of-day review. Their context: high-stakes decisions, time pressure, information density. The job: see everything clearly, log fast, stay in control.

### Brand Personality
**Sharp, focused, powerful.**

The app should feel like a professional tool built for serious people — not a consumer finance app, not a toy. It earns trust through precision and restraint. Every element is intentional.

### Aesthetic Direction
- **References**: Figma, Notion, Arc Browser — modern productivity tools that feel alive, polished, and personality-driven without being flashy.
- **Anti-references**: Robinhood (too casual), Bloomberg Terminal (too utilitarian), crypto dashboards (too loud).
- **Theme**: Dark mode only. Deep navy/space-black backgrounds (`#070810`, `#0d1018`). Per-page accent colours (indigo, sky, amber, green, teal, red, purple, orange, pink) provide identity without chaos.
- **Typography**: DM Sans for UI, JetBrains Mono for all numbers and financial data. Type should be crisp and hierarchical.
- **Motion**: Purposeful. Framer Motion stagger on page load, subtle transitions. Never distracting — motion serves comprehension, not spectacle.
- **Cards**: Glassmorphism-lite. Subtle inner glow, low-opacity borders, card-shine gradients. Elevated but not overdone.

### Design Principles
1. **Data first, decoration second.** If a visual element doesn't help the user understand or act, it shouldn't exist.
2. **Consistent rhythm.** Spacing, type size, and corner radius follow a clear scale. Pages should feel like they belong to the same family.
3. **Colour has meaning.** Accent colours are per-page identifiers. Profit = green, loss = red, warn = amber — never reassigned.
4. **Density with breathing room.** Information-rich layouts, but not cramped. The dark background does the heavy lifting; content doesn't need padding to feel spacious.
5. **Interactions feel instant.** Hover states, transitions, and feedback should make the app feel responsive and alive — like a tool that respects your time.

## Tech Stack
- Tauri 2 + React 18 + TypeScript + Vite
- Tailwind CSS 3 with extended design tokens (`bg-*`, `tx-*`, `accent-*`, `border-*`)
- Framer Motion for animation
- Recharts for data visualisation
- Lucide React for icons
- Radix UI primitives (Progress, Separator)
- DM Sans + JetBrains Mono (Google Fonts)
