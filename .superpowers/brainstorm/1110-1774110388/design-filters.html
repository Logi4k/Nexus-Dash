How it works technically

One <FilterBar> component — accepts a config array. Adding a new filter = one line in the page's config. No new component needed.
URL-synced state — active filters persist in the URL hash so refreshing or sharing preserves the filter state.
Result summary bar — shows "X of Y results" + live aggregate stats (win rate, total P&L, etc.) beneath the filter row.
Mobile — filters collapse into a single "Filter" button that opens a bottom sheet. Active filter count shown as a badge.
Accent-aware — inherits the page's colour token automatically from PAGE_THEMES. No hardcoded colours.
